# MeteorXIV Core v1.2 + EchoGate Launcher v1.2

This release is a working-state checkpoint for the current local playtest branch. It keeps the stable `v1.1.0` branch untouched while publishing the newer launcher, runtime, tutorial, zone-change, battle, and documentation work for broader testing.



EchoGate Launcher v1.2 and MeteorXIV Core v1.2 are ready for testing.

Highlights:

- New EchoGate home layout, server presets, runtime flow, launch progress, and native FFXIV settings window.
- Mac/Linux launch now uses the direct helper path instead of the Wine virtual desktop wrapper.
- Localhost is now the default server preset, with the public Demi Dev Unit server placeholder disabled until the VPS is ready.
- FFXIV settings can be edited through a native launcher window instead of the old `ffxivconfig.exe` path that crashes on Wine.
- Tutorial/session recovery is much better: stale opening/tutorial states can restore to the last known restore point instead of breaking the character.
- Public zone changes now perform a real zone transition instead of leaving the client in stale actor state.
- Battle NPC loading, respawn, action lists, spells, and trace diagnostics have been improved for enemy restoration work.
- New evidence docs were added so future enemy/stat work stays grounded in client files, traces, repo data, and original-era sources instead of guesses.

Known active issues:

- macOS windowed-mode input can still be lost after moving/minimizing the Wine game window.
- Chocobo porter content is not restored yet.
- World enemy population is still sparse and should not be treated as final 1.23b data.
- Combat formulas and many enemy action/stat details are still provisional.

As always, release packages do not include FFXIV client files, patch files, torrents, Wine runtimes, or `Data/staticactors.bin`.

## Added

- Native EchoGate FFXIV settings window for editing client config files without launching the legacy Square Enix config app.
- Graphics/settings model for FFXIV client config files, including resolution/window mode handling.
- Runtime installation/catalog plumbing for approved runtime archives.
- First-run runtime setup prompt for Mac/Linux when no approved runtime is available.
- Server preset dropdown with `Localhost`, disabled `Demi Dev Unit Server`, and `Custom Server Setup`.
- Home tab Discord link to `https://discord.gg/9w4Bjqu3Tj`.
- FFXIV settings launch/status progress so users can see work happening before the settings window appears.
- New client stat/evidence tooling and source ledgers for 1.23b stat, battle, patch-note, and enemy-population research.
- Class-scoped attribute and base-stat SQL seed tables for the new stat restoration work.
- Battle NPC action-list loading from SQL skill/spell list tables.
- Battle NPC lifecycle trace events for death, despawn, respawn readiness, respawn spawn, and skipped respawn.
- Session/zone diagnostics for logout, zone cleanup, and missing-zone edge cases.

## Changed

- Launcher branding is now `EchoGate Launcher v1.2`.
- Server startup/service labels are now `MeteorXIV Core v1.2`.
- EchoGate Home was rebuilt into a fixed, edge-anchored layout with a right-side account/status column, bottom news area, and minimum window size guard.
- Tabs that can exceed the visible area now use vertical scrolling instead of forcing manual resize.
- Runtime detection is now opinionated around approved Wine Stable or an explicit custom runtime path.
- GPTK, Whisky, and CrossOver are no longer surfaced as normal/default runtime choices in this pass.
- The Mac/Linux client path keeps the direct helper launch that successfully reaches the title/game flow.
- Windows helper selection is limited to Automatic, 32-bit, and 64-bit helpers.
- Local testing is now named `Localhost` instead of using Demi Dev server wording.
- Client patch UI is now user-provided patch-library-only. The old user-facing Download Patches button is removed.
- Manual GitHub release workflow defaults now target `v1.2` and `release-v1.2`.

## Fixed

- Removed the Wine virtual desktop wrapper path that caused client launch regressions.
- Fixed launcher Home resize overlap and narrow text wrapping issues.
- Fixed stale header/footer duplication on the Home tab.
- Fixed native settings save/create paths so config files can be recreated without using the crashing legacy config UI.
- Fixed public zone change behavior so walking between zones triggers proper zone handoff and actor reload.
- Fixed private-area actor ID collisions that could hide required opening actors during tutorial/cutscene flow.
- Added tutorial restore-point handling so interrupted opening content can resume from a safer known state.
- Improved Ul'dah opening/tutorial progression so steps wait for the relevant player action instead of advancing only by timer.
- Fixed spell action handling enough for Stone to cast, damage, apply effects, and kill the tutorial enemy in playtests.
- Fixed normal startup-loaded battle NPCs to apply respawn time, modifier lists, and stat recalculation consistently with direct spawn paths.
- Improved battle NPC names/diagnostics so missing or custom display names are easier to trace.
- Improved logout/quit cleanup ordering and session-end confirmation to reduce return-to-character-screen errors.

## Updated Documentation

- Added the leveling/stat source-of-truth ledger.
- Added the patch-note source index.
- Updated known enemy documentation with an enemy restoration evidence standard.
- Documented that `staticactors.bin` is useful for static actor paths and command/action clues, but is not proof of monster coordinates.
- Documented that temporary battle NPC fixtures must stay marked as `Test-only` or `Hypothesis` until client and trace evidence supports them.

## Known Issues

- macOS Wine window focus/input loss after dragging or minimizing the game window is still unresolved.
- Chocobo porter scripts are incomplete and may log missing script errors.
- Enemy population is not restored for full field zones yet.
- Many battle formulas, enemy stats, enemy skills, aggro rules, and action choices still need evidence.
- Some launcher runtime catalog endpoints exist as service contracts, but hosted approved runtime archives are not part of this release package.
- No FFXIV client files, patch files, torrents, Wine runtimes, or `Data/staticactors.bin` are included.

## Tester Notes

- Use the `Localhost` server preset for current local testing.
- Prepare `Data/staticactors.bin` from your own FFXIV 1.x client folder before running Map Server.
- Use the native EchoGate FFXIV settings window instead of the legacy `ffxivconfig.exe`.
- Report launch crashes with the EchoGate launch log, helper log, and Map/Lobby/World terminal output when possible.
