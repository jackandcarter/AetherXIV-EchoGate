# AetherXIV Core v1.3 + EchoGate Launcher v1.3

This release focuses on launcher polish, setup reliability, the first Umbra
framework distribution path, and stronger opening-tutorial playtest behavior.

## Highlights

- EchoGate Launcher v1.3 with refreshed Home, Server, Client, Umbra, Runtime,
  and Launch Log flows.
- Aether Umbra framework bundle artifact for early framework testing.
- Launcher news admin page for creating, editing, publishing, hiding, and
  deleting `/launcher/news` posts.
- Improved macOS, Linux, Windows, and Wine setup guidance.
- Updated server-core packaging under the AetherXIV name.
- Gridania opening tutorial battle flow now uses director-owned companion
  cleanup, action-gated enemy mortality, and seeded companion battle action
  lists for Yda and Papalymo.
- Dev diagnostics now use the `AETHER_DEV_DIAGNOSTICS*` names, with legacy
  `METEOR_DEV_DIAGNOSTICS*` still accepted only for compatibility.

## Umbra Status

Umbra is still an opt-in testing path. Framework installation, launch-time
injection settings, repository/catalog plumbing, bottom-right toast panels,
system-plugin startup, Dev Bridge control, and the in-game shell are active
development areas. Third-party plugin execution remains disabled by default.

## Database Migrations

Existing self-hosted databases should run:

```sh
./tools/apply-db-migrations.sh
```

This applies the launcher/Umbra service tables and the Gridania tutorial
companion action-list data without dropping existing character or account data.

## News Admin

Local admin URL:

```text
http://127.0.0.1:8080/launcher/admin/news
```

Remote access requires `AETHER_LAUNCHER_ADMIN_PASSWORD` or a launcher admin
password configured in `Data/www/launcher/config.local.php`. Legacy
`METEOR_LAUNCHER_ADMIN_PASSWORD` remains supported as a fallback.

## Release Assets

Expected assets include EchoGate builds for Linux, macOS, and Windows,
`AetherXIV-Server-Core-v1.3`, and `AetherXIV-Umbra-Framework-v1.3`.
